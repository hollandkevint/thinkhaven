# Prompt Kit: AI Communication (Working with AI Agents Makes You a Better Communicator)
*By Nate Jones*
Source: https://natesnewsletter.substack.com/

## Summary

The kit operationalizes the six-field brief framework from Nate's companion article — goal, context, sources, constraints, quality bar, definition of done — and turns it into three standalone prompts a user can run in any AI tool.

The three prompts are different entry points into the same discipline. **Useful Question Builder** walks the user through all six fields conversationally and produces a complete brief from scratch. **Vague Ask Auditor** takes an existing request, diagnoses what's missing or ambiguous across the six fields, then rewrites it. **Definition-of-Done Generator** articulates what "finished" looks like so work doesn't drift, race ahead, or produce something that looks right but doesn't work.

All three work in ChatGPT, Claude, or Gemini without setup. The shared tonal pattern is "do not start doing the user's actual task." The scaffold is the deliverable. Most prompt kits race to execute; this one refuses to, by design.

## Mapping to Thinkhaven

This kit is the closest external artifact to what Thinkhaven's three modes already do — but as standalone prompts a user must run themselves.

**1. Useful Question Builder is what Assessment mode should feel like.** Both walk the user through building a complete brief before the AI starts working. Assessment mode can opinionate the sequencing and pacing in a way a flat prompt cannot.

**2. Vague Ask Auditor maps to a Stress Test pattern.** Take an existing artifact, find what's missing, force the user to confront gaps before more work piles on top. Thinkhaven's Stress Test mode does this conversationally; the prompt does it transactionally.

**3. Definition-of-Done Generator is the executive-prep pattern.** Nate names "definition of done" as where most work drifts. Executive Prep mode encodes this — the artifact has to survive a stakeholder room, so the definition of done is the room itself.

**4. Strategic implication: the prompts exist because the discipline is hard.** Nate published these because the briefing pattern doesn't happen by default — users need a scaffold. Thinkhaven's bet: the scaffold should be the product, not a prompt the user copy-pastes into a blank chat.

## Implication for the roadmap

The three prompts give Thinkhaven a clean benchmark for mode quality: a session in Assessment should produce a brief at least as complete as Useful Question Builder's output, without the user having to know the six fields. If a Thinkhaven session ends with a thinner brief than the prompt would have produced, the mode is underperforming. Worth running side-by-side validation: take a real user task, run it through Useful Question Builder, then through Assessment mode, compare outputs.
